# 子 Agent 团队说明

本项目用 Claude Code 子 agent 模拟一个小型研发团队。所有 agent 共享同一个工作区和仓库,**无需开多个 VSCode**。主对话窗口里的 Claude 充当"主调度",负责把任务派发给下面的角色,角色干完向主调度回报。

| 角色 | 名字 | 职责 |
|------|------|------|
| 项目经理 / 架构师 | **elon** | 澄清需求、拆解架构、定义接口、产出任务卡与里程碑(不写功能代码) |
| 开发工程师 | **tom** | 按任务卡实现代码(未来可扩展 ken/Xill 等更多开发) |
| 测试工程师 | **gill** | 对照验收标准独立交叉验证 Tom 的实现 |

## 协作链路
1. 你 ↔ **elon** 谈需求 → elon 在 `docs/` 沉淀需求/架构/任务卡。
2. 主调度按任务卡派给 **tom** 实现。
3. 主调度把实现交给 **gill** 验证,gill 出验证报告。
4. 有问题回到 tom 修,或回到 elon 重新决策。

## 怎么调用
- 显式点名:在对话里说"让 elon 来规划这个需求""派 Tom 实现任务卡 01""让 Gill 验证一下"。
- 主调度也会在任务匹配时自动选择合适的 agent。

## 扩展开发团队
未来加开发时,复制 `tom.md` 改名即可,例如 `ken.md`(控制模块)、`xill.md`(SoC 驱动),在 description 里写清各自负责的模块边界。

# 架构文档：ROS2 ↔ micro-ROS 串口双向最小闭环

> 第一个需求的总体架构。作者：elon（设计）。本文是开发(Tom)与测试(Gill)的设计依据，接口细节以 `01-接口契约.md` 为准。

## 1. 目标
在 WSL(Ubuntu 24.04) 跑 ROS2 **Jazzy**，在 STM32 **Nucleo-F401(STM32F401)** 跑 **micro-ROS**，二者通过**串口(921600 + DMA，双向)** 连接，打通**最小双向 pub/sub 闭环**。

现实边界：用户当前**无 STM32 硬件**。因此工作分两阶段：
- **Phase A（无硬件即可完成）**：ROS2 侧节点/消息、micro-ROS Agent 起法、固件工程骨架(能编译出 .elf/.bin)、构建脚本——全部就绪。
- **Phase B（需真机）**：把 Nucleo 用 USB 接电脑，agent 串口连通，验证双向 topic 回环。

## 3. 数据流（最小闭环 topic 设计）

双向各一条，先用最朴素的 `std_msgs` 打通链路、验证「双向 + 回环正确性」；闭环稳定后再换自定义 `exo_msgs`。

| 方向 | topic | 类型 | 频率 | 语义 |
|------|-------|------|------|------|
| WSL → MCU | `/exo/cmd_heartbeat` | `std_msgs/msg/Int32` | 10 Hz | WSL 发递增计数器(心跳) |
| MCU → WSL | `/exo/mcu_status` | `std_msgs/msg/Int32` | ~10 Hz | MCU 回显收到的计数器值(回环校验) |

**回环正确性判据**：MCU 把收到的 `cmd_heartbeat` 值原样回填到 `mcu_status`。WSL 侧比对发出值与收回值一致 ⇒ 双向链路通。详见接口契约。

## 4. 串口 / USART 引脚选型（已查证 Nucleo-F401RE 用户手册级事实）

**选定：USART2 = PA2(TX) / PA3(RX)。**

理由：Nucleo-F401RE 板载 ST-Link 出厂默认（焊桥 SB13/SB14 ON）把 **USART2 接到 USB 虚拟串口(VCP)**。因此**一根 USB 线**即可把 MCU 的 USART2 透传给 WSL（在 WSL 里表现为 `/dev/ttyACM0`），**无需额外 USB-TTL 转接板**——对「只有一块板、无外设」的现实边界最友好。

**备选：USART6 = PA11(TX) / PA12(RX)（走 CN10）**，仅当后续要把 VCP 留给日志打印、改用独立 USB-TTL 接 micro-ROS 时启用。届时需把 agent 的 `--dev` 指向对应的 `/dev/ttyUSB*`。

**串口参数**：波特率 **921600**，**8N1**，**TX/RX 双向 DMA**（DMA 优先级 Very High），符合 ST / micro-ROS 官方 serial-with-DMA transport 推荐。

> WSL 注意：USB 串口设备需通过 `usbipd-win` 从 Windows attach 到 WSL 才会出现 `/dev/ttyACM0`。此为 Phase B 真机联调步骤，记在 Tom 的 T7 / Gill 的 G7。

## 5. micro-ROS Agent 连接方式（已查证）

- snap 版 agent 的 serial 传输有已知问题，**采用从源码 build（`micro_ros_setup`，jazzy 分支）或 Docker**。
- 启动命令形如：
  ```bash
  ros2 run micro_ros_agent micro_ros_agent serial --dev /dev/ttyACM0 -b 921600 -v6
  ```
- 传输层：**serial (XRCE-DDS over serial)**，agent 运行在 WSL 侧。
